# Pressure Test: Blank Page, Low Context

## User request

Write a great announcement for my product launch.

## Expected behavior

- Do not block unless required.
- State assumptions or ask one focused question if product/audience is essential.
- If proceeding, create a fill-in scaffold or draft with placeholders for missing human/product detail.
- Mark places where real detail is needed.

## Good response traits

- Infers `blank_page`, standard or quick depending phrasing.
- Does not invent product features.
- Provides useful structure and placeholders.
- Flags missing details such as product name, audience, launch benefit, proof, CTA.
