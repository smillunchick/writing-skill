# Mode Routing Details

## Blank page

Use when the user gives a topic, goal, or format but little prose.

Workflow:
1. Infer audience, purpose, format, desired outcome, emotional temperature, and constraints.
2. If necessary, ask one focused clarification. Otherwise state assumptions briefly.
3. Generate raw material before polished prose: possible angles, claims, examples needed, likely reader objections.
4. Draft with the chosen depth.
5. Run AI-slop/human-specificity pass.
6. Stop with final draft plus missing-detail notes if needed.

Missing-detail note:

```markdown
## Places to add real detail
- [Insert a specific example here]
- [Add what you actually think about X]
- [Add a concrete moment, number, or observation]
```

## Notes to draft

Use when the user provides bullets, notes, transcript fragments, or rough ideas.

Workflow:
1. Preserve the user’s actual claims and useful phrasing.
2. Cluster notes into sections.
3. Identify the throughline.
4. Draft in the requested format.
5. Keep idiosyncratic details; do not over-normalize.
6. Remove generic smoothing.
7. Stop with final draft and short notes.

## Revise

Use when the user provides an existing draft and requests improvement.

Workflow:
1. Identify revision target: clearer, shorter, warmer, sharper, more direct, more persuasive, more personal, more natural, less corporate, less AI-ish, more literary, more executive, more emotionally restrained, or more memorable.
2. If targets conflict, prioritize audience and stated goal.
3. Identify weak spans.
4. Rewrite surgically unless the whole structure is broken.
5. Stop with revised version and concise change notes.

## Audit

Use when the user wants diagnosis rather than immediate rewriting.

Workflow:
1. Give a verdict.
2. Identify what works.
3. Identify highest-impact fixes.
4. Provide span-specific critique.
5. Rewrite only if requested or obviously useful.

## Finish

Use for final polish/proofread.

Workflow:
1. Preserve meaning and voice.
2. Fix grammar, punctuation, repetition, awkward rhythm.
3. Avoid making the text more generic.
4. Stop with ready-to-send version.
